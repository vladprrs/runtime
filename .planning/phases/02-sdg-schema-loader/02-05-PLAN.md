Close the one remaining gap in Phase 2 verification: implement edge type-compatibility checking in the DAG validation pass (Pass 4).

Purpose: D-28 explicitly requires Pass 4 to "type-check all edges -- output type of source must match expected input type of target port." The TypeMismatch and DagInvalidPort error variants are defined but never raised. Without this, invalid SDGs where e.g. a string output feeds an integer-expecting input pass validation silently and would only fail at runtime in Phase 4. This contradicts SC3's promise that invalid definitions are caught at load time.

    **Step 3: Define output type resolution.**
    Create `fn resolve_output_type(node: &ComputationNode, definition: &ServiceDefinition) -> Option<String>` that returns the SDG type string for a node's output:
    - `field`: Look up `params["name"]` in aggregate fields (check all aggregates; also check IMPLICIT_FIELDS). Return `field_type`. If not found, return None (semantic pass already catches this).
    - `command`: Look up `params["name"]` in transition command fields across all aggregates. Return field_type. If not found, return None.
    - `context`: Look up `params["path"]` in the VALID_CONTEXT_PATHS table. Return the type. If not found, return None (semantic pass catches this).
    - `literal`: Return `params["output_type"]` as string. If missing, return None (semantic pass catches this).
    - `lookup`: Return the picked field type (params["pick"] in params["aggregate"]'s fields, including implicit fields). Could be complex to resolve; if unresolvable, return None and skip type-checking for that edge.
    - `lookup_many`: Same as lookup but wrap result as `T[]` (e.g., "string" -> "string[]").
    - `filter`: Output is same type as input (T[] -> T[]), resolved from incoming edge. For simplicity at validation time, return None (passthrough -- skip type-checking on output edges from filter).
    - `count`, `str_len`, `length`: Return "integer".
    - `sum`, `add`, `sub`, `mul`, `div`: Return "number".
    - `any`, `all`, `eq`, `neq`, `gt`, `lt`, `gte`, `lte`, `and`, `or`, `not`, `contains`, `str_contains`: Return "boolean".
    - `min`, `max`: Passthrough type from input, return None.
    - `concat`: Return "string".
    - Unknown type: Return None.

    **Step 4: Implement type compatibility check.**
    Create `fn is_type_compatible(source_type: &str, port_type: &PortType) -> bool`:
    - `Exact(expected)`: source_type == expected. Special case: "integer" and "float" are both compatible with "number".
    - `Array(expected)`: source_type must end with "[]" AND the element type must match expected. "number[]" matches Array("number"), "integer[]" and "float[]" also match Array("number").
    - `AnyArray`: source_type must end with "[]".
    - `Any`: always true (generic type parameter).
    - `Variadic(expected)`: same as Exact(expected).
    - `StringOrArray`: source_type == "string" OR source_type ends with "[]".

    **Step 5: Wire into materialize_dags.**
    After the successful `toposort`, add an edge validation loop. Build a `HashMap<String, &ComputationNode>` for O(1) node lookup. For each edge:
    1. Look up source node and target node by edge.from / edge.to.
    2. Get the valid ports for the target node type. If unknown type, skip (semantic pass handles).
    3. Check if `edge.port` (accounting for "in" with index on variadic ports) is a valid port name. If not, push `SdgError::DagInvalidPort { port: edge.port, node: edge.to, node_type: target.node_type }`.
    4. If port is valid, resolve the source node's output type. If resolved to Some(type), check compatibility with the port's expected type. If incompatible, push `SdgError::TypeMismatch { path: format!("computations.edges[{from} -> {to}.{port}]", from=edge.from, to=edge.to, port=edge.port), expected: expected_type_description, found: source_type }`.
    5. If source type is None (unresolvable -- e.g., field node referencing unknown field), skip type-checking for that edge (semantic pass already catches the root cause).

    Collect all type errors and return them if non-empty. If empty, return Ok(MaterializedDag).
