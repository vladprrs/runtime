**sdg_schema.json** -- Create a comprehensive JSON Schema (Draft 2020-12) at `crates/sdg-loader/src/sdg_schema.json`. Per D-01, this covers ALL 6-pager sections. Per D-05, single monolithic schema with internal `$defs`.

The schema MUST define:
```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "$id": "https://runtime.dev/sdg/v1",
  "title": "Service Definition Graph",
  "description": "Schema for SDG files consumed by the Runtime Execution Layer",
  "type": "object",
  "required": ["schema_version", "service", "aggregates", "transitions"],
  "properties": {
    "schema_version": { "type": "string", "pattern": "^\\d+\\.\\d+\\.\\d+$" },
    "service": { "$ref": "#/$defs/ServiceMeta" },
    "glossary": { "$ref": "#/$defs/Glossary" },
    "aggregates": { "type": "object", "additionalProperties": { "$ref": "#/$defs/Aggregate" }, "minProperties": 1 },
    "transitions": { "type": "object", "additionalProperties": { "$ref": "#/$defs/Transition" }, "minProperties": 1 },
    "projections": { "type": "object", "additionalProperties": { "$ref": "#/$defs/Projection" } },
    "endpoints": { "$ref": "#/$defs/Endpoints" },
    "external_dependencies": { "type": "object" },
    "processes": { "type": "object" }
  },
  "$defs": { ... }
}
```

The `$defs` section MUST include definitions for:
- `ServiceMeta` -- required: name (string), description (string), owner (string)
- `Glossary` -- terms object of string values
- `Aggregate` -- required: fields (object of FieldDef), states (array of strings, minItems 1), initial_state (string)
- `FieldDef` -- required: type (string enum: "string", "integer", "boolean", "float", "date", "datetime", "uuid"); optional: required (boolean), default (any)
- `Transition` -- required: aggregate (string), from (string), to (string), command (CommandDef), events (array of strings, minItems 1); optional: guard (GuardDef)
- `CommandDef` -- required: fields (object of FieldDef)
- `GuardDef` -- required: nodes (object of DagNode), edges (array of 2-element string arrays), output (string)
- `DagNode` -- discriminated union on "type" field using `oneOf` with `if/then` or `discriminator`. Types: field_access, comparison, boolean_logic, arithmetic, string_op, literal, ts_block, decision_table, integration_call. Each variant has type-specific required properties:
  - field_access: required field (string), optional path (array of strings)
  - comparison: required op (enum: eq, neq, gt, lt, gte, lte), left (string), right (string)
  - boolean_logic: required op (enum: and, or, not), operands (array of strings)
  - arithmetic: required op (enum: add, sub, mul, div), left (string), right (string)
  - string_op: required op (enum: concat, contains, length), operands (array of strings)
  - literal: required value (any)
  - ts_block: required source (string), optional config (object) -- stub per D-01
  - decision_table: optional config (object) -- stub per D-01
  - integration_call: optional config (object) -- stub per D-01
- `Projection` -- required: type (string), source_aggregate (string); optional: fields (array of strings), group_by (string), aggregation (string)
- `Endpoints` -- optional: commands (object mapping route strings to CommandEndpoint), queries (object mapping route strings to QueryEndpoint)
- `CommandEndpoint` -- required: transition (string)
- `QueryEndpoint` -- required: projection (string)

**CRITICAL:** The DagNode discrimination in JSON Schema must use `oneOf` with each option requiring `"type": {"const": "field_access"}` (or the specific type). This is the standard pattern for tagged unions in JSON Schema Draft 2020-12.

**schema.rs** -- Implement the schema module per D-02 (embedded in binary):
```rust
use jsonschema::Validator;
use crate::error::SdgError;

const SDG_SCHEMA_JSON: &str = include_str!("sdg_schema.json");

pub fn create_validator() -> Validator {
    let schema: serde_json::Value = serde_json::from_str(SDG_SCHEMA_JSON)
        .expect("embedded schema must be valid JSON");
    jsonschema::draft202012::new(&schema)
        .expect("embedded schema must be valid Draft 2020-12")
}

pub fn validate_schema(instance: &serde_json::Value) -> Vec<SdgError> {
    let validator = create_validator();
    validator.iter_errors(instance)
        .map(|error| SdgError::SchemaViolation {
            instance_path: error.instance_path.to_string(),
            schema_path: error.schema_path.to_string(),
            message: error.to_string(),
        })
        .collect()
}
```

Note: Check the exact API for `instance_path` and `schema_path` on `ValidationError` -- they may be accessed as `.instance_path` (a `JsonPointer`) and `.schema_path` (a `JsonPointer`). Both implement `Display`. If the API differs, adapt accordingly. The key constraint is converting to owned Strings immediately to avoid lifetime issues (Pitfall 1 from research).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File system to SDG loader | SDG file read from disk; content is untrusted until validated |
| JSON parser | Raw bytes become structured data; malformed input could cause issues |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
